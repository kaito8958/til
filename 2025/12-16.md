# TIL（2025-12-16）

## jQuery の submit イベントでの落とし穴を学んだ
- jQuery のイベントハンドラでは、コールバックが `false` を返すと送信処理がキャンセルされる仕様がある。
- CoffeeScript では代入式が値を返すため、`@dirty = false` のような記述が意図せず `false` を返し、フォーム送信を止めてしまうケースがある。
- submit イベント内では、明示的に `return` を書くことで送信を妨げないようにする必要がある。

## Network タブを使った原因切り分けの重要性を学んだ
- 保存ボタンを押しても Network にリクエストが出ていない場合、バックエンドではなくフロントエンド側で処理が止まっていると判断できる。
- 「何が起きていないか（リクエストが飛んでいない）」を確認することで、原因を大きく絞り込めることを実感した。

## ステージング環境でのデプロイと CI/CD の流れを体験した
- ブランチに push することで CI/CD が自動実行され、ステージング環境へ反映される仕組みを実際に確認できた。
- `make staging-deploy` はローカルの変更を直接反映するものではなく、GitHub 上のブランチ状態を元にデプロイされることを理解した。
