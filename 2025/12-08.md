# TIL（2025-12-08）

## 1. Docker 環境で JS が動かない原因を特定し、npm ビルドで解決できた

Tech-Master のカリキュラム編集画面で、Markdown プレビューが表示されない問題を調査した。

ブラウザのコンソールを見ると、以下のエラーが大量に発生していた：

- `bundle.admin.js` が 404  
- `jQuery is not defined`  
- 依存している JS が全体的に読み込まれていない

### ▶ 原因  
**Docker コンテナ内で JS ビルドが実行されていなかった。**  
このプロジェクトは古い Webpack を使っており、  
Rails 側とは別に、コンテナ内の npm で JS をビルドする必要があった。

### ▶ 実際に行った操作

```bash
docker exec -it tech-master-web-1 bash
npm install
npm run build
この操作によって `bundle.admin.js` が生成され、プレビューが正常に表示された。

### ▶ 学び  
- JS が 404 のときは「ビルドされていない可能性」をまず疑う  
- Docker 環境では **ローカルではなく “コンテナ内” の node/npm を使う** 必要がある  
- Rails が表示されていても、JS 側が正しく準備されていないと画面は正常に動かない

## 2. 問題再現と Issue アウトライン作成を通じて、実務の進め方を掴んだ

初めての実務風プロジェクトで、以下の流れを経験した：

- 管理画面にアクセスし、問題を自分で再現する  
- エラー内容から原因を切り分け  
- 必要なファイルの場所や構造を確認  
- 実際に Issue のアウトラインを作成し、Slack でレビュー依頼した  

### ▶ 学び  
- 実務では「何をどこまで把握したか」をアウトラインとして整理し、  
  **レビュー前でも不明点を残したまま提出してOK** という進め方が自然  
- Slack に GitHub URL を貼ると、自動プレビューされる仕組みも理解した  
- 問題再現 → 原因調査 → アウトライン作成 → レビュー依頼  
  という一連の流れを体験できた
